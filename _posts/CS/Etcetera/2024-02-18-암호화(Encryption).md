# ☻ 암호 알고리즘

데이터 무결성, 기밀성 확보를 위해 정보를 쉽게 해독할 수 없는 형태로 변환하는 기법이다. 암호 알고리즘은 크게 단방향 암호화와 양방향 암호화로 나뉜다.

---

## ☺︎ 단방향 암호화(해시 암호 방식)

원본 데이터를 **암호화 한 후에는 복호화가 불가능**한 암호화 방식. 주로 **해시 함수**를 사용한다. 한 번 암호화된 데이터는 원래의 데이터로 복구할 수 없다.

### 해시함수와 해시값?

해시 함수는 **임의의 길이의 데이터를 고정된 길이의 데이터로 매핑**하는 함수이다. 주로 패스워드 저장, 데이터 검색 등에 활용된다. 해시 함수는 **동일한 입력 값에 대해 항상 동일한 출력 값을 제공**하며, 일반적으로 원본 데이터를 복구하는 것은 불가능하다.

해시 값은 **해시 함수를 통해 얻어진 고유한 값**으로, 원본 데이터로부터 일정한 길이의 문자열을 생성한다.

### 해시함수는 왜 복호화가 불가능하지

해시 함수는 **'일정한 길이의 입력을 받아서 고정된 길이의 해시값을 출력하는 함수'** 이다. 이 과정에서 입력값이 아무리 크더라도 출력되는 해시 값의 길이는 항상 일정하다. 이 말은 즉, 무한히 많은 가능한 입력값들이 한정된 수의 해시값들로 매핑된다는 것이다.

> 예를 들어
>

"안녕하세요"라는 문장을 해시 함수를 통해 변환하면 "abc123"라는 해시값을 얻을 수 있다. 여기서 "안녕하세요"는 원본 데이터(입력), "abc123"는 해시값(출력)이다.

그런데 해시 함수는 **'단방향' 함수**라서, 한번 해시값으로 변환된 데이터는 다시 원본 데이터로 돌릴 수 없다. 즉, "abc123"으로부터 "안녕하세요"를 알아낼 수 없다는 뜻이라고 한다.

### 왜 단방향 함수는 복호화가 불가능하지

단방향 함수는 **입력값을 특정한 규칙에 따라 변환하여 출력값을 생성하는 함수**인데, 이때 생성된 출력값으로부터 **원래의 입력값을 찾아낼 수 없는 구조**로 되어 있다.

단방향 함수는 **정보를 압**축하는 과정을 거치는데, 이 과정에서 원래의 정보가 **일부 손실**되기 때문에 **복호화가 불가능**하다. 이는 고유한 해시 값을 생성하기 위해 해시 함수가 원래 데이터의 각 부분을 복잡하게 섞고, 이를 통해 완전히 새로운 값을 생성하는 방식 때문 이라고 한다.

---

## ☺︎ 양방향 암호화

원본 데이터를 암호화한 후 **복호화를 통해** **원본 데이터로 다시 변환할 수 있는** 암호화 방식이다. **대칭키와 공개키 암호화 방식**이 이에 해당한다.

---

## 암호 알고리즘의 방식

## 1. 양방향

### 1. 대칭키 암호화 방식

대칭키 암호화 방식은 **암호화와 복호화에 같은 키**를 사용하는 방식이다. 이 방식은 암호화와 복호화 과정이 빠르다는 장점이 있지만, 키를 안전하게 전달하는 문제가 있다.

### 종류

| 암호화 알고리즘 | 설명 |
| --- | --- |
| DES (Data Encryption Standard) | 미국 표준 기술 연구소(NIST)에서 제정한 암호화 표준. 블록 크기는 64, 키 길이는 56인 페이스텔 구조 |
| SEED | 한국정보보호진흥원에서 개발하였다. 블록 크기는 128비트이며 키 길이에 따라 128, 256 비트로 분류  |
| AES (Advanced Encryption Standard) | 미국 표준 기술 연구소(NIST)에서 제정한 암호화 표준. 블록 크기는 128이고, 길이는 128, 192, 256비트의 키 길이를 지원하며, 블록 암호화 방식을 사용한다.  |
| ARIA | 한국정보보호진흥원에서 개발했다. 블록 크기는 128비트이며 128, 192, 256비트의 키 길이를 지원하는 블록 암호화 알고리즘. 한국의 국가 표준 암호 알고리즘으로, 다양한 암호화 상황에 사용된다. |
| IDEA (International Data Encryption Algorithm) | 스위스의 암호화 연구소에서 개발한 128비트 키를 사용하는 블록 암호화 알고리즘. 높은 보안성을 가지고 있으며, 주로 유럽에서 사용된다. |
| LFSR (Linear Feedback Shift Register) | 피드백 회로를 가진 시프트 레지스터를 이용한 암호화 알고리즘. 스트림 암호화 방식에 주로 사용되며, 암호화와 복호화가 빠르다는 장점이 있다. |

### 128비트 256비트 암호의 차이는 무엇인가

즉, 128비트 암호화 키는 2의 128승 개의 가능한 키 중에서 하나를 선택하는 것과 같다. 비슷하게, 256비트 암호화 키는 2의 256승 개의 가능한 키 중에서 하나를 선택하는 것과 같다. 따라서 비트 수가 많을수록 암호화 키의 선택 범위가 넓어져서 더 높은 보안성을 갖는다고 한다.

---

### 2. 비대칭키 암호화 방식

비대칭키 암호화 방식은 **암호화와 복호화에 서로 다른 키를 사용하는 방식이다.** 두 개의 키 중 **하나는 공개키로 공개**되고 다른 **하나는 비밀키**로 비밀리에 보관된다. 이 방식은 대칭키 암호화 방식의 키 전달 문제를 해결하지만, 암호화와 복호화 과정이 대칭키 암호화 방식보다 느리다.

| 암호화 알고리즘 | 설명                                                                         |
| --- |----------------------------------------------------------------------------|
| RSA | 공개키(비대칭) 암호화 방식 중 하나. MIT 수학 교수진이 고안했으며, 큰 인수의 곱을 소인수 분해하는 수학적 알고리즘을 사용했다. |
| 디피-헬만 | 키 교환 알고리즘으로, 안전하게 통신하는 두 개체가 공유 비밀을 생성하는 방법을 제공한다.                         |
| ECC (Elliptic Curve Cryptography) | 타원곡선 위의 점에 대한 대수 문제의 어려움을 이용한 암호화 방식이다.                                    |
| ElGamal | 공개키 암호화 시스템 중 하나로 디피-헬만 키 교환을 기반으로 한다.                                     |

---

## 2. 일방향

### 해시 함수

해시 함수는 원본 메시지를 암호화하되, 복호화는 불가능한 방식이다. 이 방식은 무결성 검사나 패스워드 저장 등에 주로 사용된다.

- 예: MD5(Message Digest Algorithm 5), SHA(Secure Hash Algorithm) 계열, HAS-160, HAVAL 등

---

### Appendix

스프링에서 `PasswordEncoder` 인터페이스를 사용한 적이 있다.

스프링 프레임워크는 암호화를 쉽게 구현할 수 있도록 `PasswordEncoder`라는 인터페이스를 제공한다. 이 `PasswordEncoder` 인터페이스를 구현하는 여러 개의 클래스가 제공되는데, 그 중에는 `BCryptPasswordEncoder`, `Pbkdf2PasswordEncoder`, `SCryptPasswordEncoder` 등이 있다. 이러한 클래스들은 각각 다른 암호화 알고리즘을 사용하여 패스워드를 암호화하고 검증한다. 내가 궁금했던 건 스프링이 만들어질 때 이런 암호화 알고리즘이 어떻게 적용됐는지 궁금했는데, 공식 문서에도 알려주질 않는다..
