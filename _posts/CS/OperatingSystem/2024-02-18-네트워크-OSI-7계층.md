# ☻ 네트워크 OSI 7계층

2024년 2월 17일

# What is OSI Model?

OSI 모델은 ISO에서 1984년 만들어졌다. 컴퓨터 ↔ 데이터 사이의 데이터 전송 흐름을  설명하는 프레임워크이다. 전문화된 네트워크 기능을 수행하기 위해 7개의 계층으로 나눠져있으며, 더 체계적인 네트워킹 접근을 가능하게 한다.

### **그래서, OSI 7 계층을 왜 나눌까?**

네트워크 문제를 트러블 슈팅하기 용이하기 때문. 문제가 있는 레이어만 수정하면 된다. 다른 레이어는 그대로 유지된다.

---

# OSI 7 Layer

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/6d607010-0c2d-4aea-a381-64382a733ad5)

| No | Layer |
| --- | --- |
| 1 | Physical Layer (물리 계층) |
| 2 | Data Link Layer (데이터 연결 계층) |
| 3 | Network Layer (네트워크 계층) |
| 4 | Transport Layer (전송 게층) |
| 5 | Session Layer (세션 계층) |
| 6 | Presentation Layer (표현 계층) |
| 7 | Application Layer (응용 계층) |

----

### 1. Physical Layer - Layer 1 (물리)

> 리피터, 케이블, 허브 등
>

단지, 데이터를 전송하는 역할만 진행한다.

OSI 계층의 가장 낮은 레이어는 물리 계층으로. 장치간의 실질적인 연결을 담당하며 
바이트 정보를 포함한다. 
또한 한 노드에서 다른 노드로 바이트를 전송하는 역할을 담당한다. 데이터를 송신하면 레이어는 신호를 받고 0s와 1s로 전환하고 데이터 연결 계층으로 보낸다.

---

### 2. DataLink Layer - Layer 2 (데이터 연결)

> 브릿지, 스위치 등
>

데이터 연결 계층은 물리적 계층을 통해 **한 노드에서 다른 노드로의 데이터 전송**에 오류가 없는지 확인한다. 이 계층에서 패킷은 ‘**프레임**’이라는 단위로 취급된다.

### 4계층인 전송 계층과 2계층인 데이터 연결 계층의 차이가 모호하다

전송 계층은 네트워크 계층에서 받은 데이터를 응용 계층으로 보내거나, 응용 계층에서 받은 데이터를 네트워크 계층으로 보내는 역할을 한다. 이 과정에서 데이터의 전송이 효율적이고 안정적이게 하기 위해 **세그먼트(segment)** 단위로 데이터를 나눈다.

반면 데이터 연결 계층은 물리 계층에서 전송된 데이터에 오류가 없는지 확인하고, 에러가 있을 경우 이를 수정하는 역할을 한다. 이 계층에서는 데이터를 **프레임(frame)** 이라는 단위로 처리한다.

즉, 주요 차이점은 전송 계층은 데이터의 전송을 관리하고 제어하는 반면, 데이터 연결 계층은 데이터의 오류 검출 및 수정과 같이 더 낮은 수준의 작업을 수행하는 것이 주요 차이점이다.

---

### 3. Network Layer - Layer 3 (네트워크)

> 라우터, IP
>

네트워크 계층은 다른 네트워크에 위치한 호스트에게 데이터를 전송하는 역할을 한다. 즉, 데이터를 목적지까지 안전하고 빠르게 전달하는 기능을 담당한다. 라우터를 통해 이동할 경로를 선택해 IP 주소를 지정하고, 해당 경로에 따라 패킷을 전달해준다.

### Packet(패킷)

패킷은 네트워크를 통해 전달되는 데이터의 가장 기본적인 논리적 중재이다. 컴퓨터 네트워크에서, 패킷은 컨테이너 또는 TCP/IP 네트워크와 인터넷 작업 사이의 데이터를 전달하는 역할을 한다.

---

### 4. Transport Layer - Layer4 (전송)

> TCP, UDP
>

전송계층은  응용 계층에게 서비스를 제공하고, 네트워크 계층으로 부터 서비스를 받아온다. 전송계층은 완전한 End to End 메시지 전달에 책임이 있다. **TCP, UDP 프로토콜을** 활용해서 통신을 활성화한다.

### TCP 프로토콜

전송 계층에 위치하며, 컴퓨터에서 실행되는 프로그램간에 일련의 옥텟을 안정적으로 교환할 수 있게 해주는 연결형 프로토콜. 흐름제어 기능을 활용해서 송신 및 수신 속도를 일치시킨다.

핵심은 **신뢰성 있는 연결 지향형 전달 서비스.** 기본 헤더 크기는 20byte → 60byte까지 확장이 가능하다. 스트림 전송 기능을 제공하고, 순서 제어, 오류 제어, 흐름 제어 기능을 제공한다.

### UDP

**비연결성이고, 신뢰성이 없으며 순서화되지 않은 데이터 그램 서비스**를 제공한다. 실시간 응용 및 멀티 태스킹이 가능하여 빠른 요청과 응답이 필요한 실시간 응용에 적합하다. 헤더는 고정 크기의 8byte만 사용한다.

### 그럼 TCP, UDP 프로토콜 둘 중 무엇을 선택해야 하는지

보았을 땐 TCP가 더 장점이 많아보인다. 그럼 TCP를 사용하면 되는 게 아닌가 싶었다. 하지만 상황에 따라 다르다.

TCP는 데이터의 신뢰성을 보장하는 프로토콜이. 데이터 패킷이 손실되거나 오류가 발생했을 경우 재전송을 통해 오류를 복구하며, 순서대로 데이터를 전송한다. 이런 특성 때문에 웹 페이지 로딩, 이메일 전송, 파일 다운로드 등 **데이터의 정확성이 중요한 경우에 주로 사용된다.**

반면에 UDP는 **신뢰성을 보장하지 않지만 데이터 전송 속도가 빠르다.** 데이터 패킷의 순서나 손실에 대해 관리하지 않기 때문에 **오버헤드가 적어 빠른 전송 속도**를 가진다. 이런 특성 때문에 실시간 스트리밍, 온라인 게임, VoIP(Voice over IP) 등 실시간성이 중요하고, 일부 패킷 손실이 허용되는 서비스에서 주로 사용된다.

즉, **'신뢰성'이란 단어는 데이터의 전송과정에서 패킷의 손실, 오류, 순서 등을 철저히 관리한다는 의미에서 사용**되며, 이런 관리 과정은 데이터 전송의 속도를 늦출 수 있다. 따라서 **실시간성이 중요한 온라인 게임이나 스트리밍 서비스에서는 일부 패킷 손실을 허용하더라도 빠른 데이터 전송**을 위해 UDP를 사용하는 경우가 많다.

---

### 5. Session Layer - Layer5 (세션)

> API, Socket
>

연결, 세션 유지, 권한부여, 안전을 보장하는 역할을 한다. 데이터가 통신하기 위한 논리적 연결을 담당한다. TCP/IP 세션을 만들고 없애는 책임을 지니고 있다.

---

### 6. Presentation Layer - Layer6 (표현)

데이터 표현 계층은 네트워크에서 전송되는 **데이터의 표현 방식을 관리**한다. 이 계층에서는 데이터를 암호화하거나 압축하는 등의 변환 작업을 수행한다. 이를 통해 데이터가 안전하게 전송될 수 있도록 보장한다.

---

### 7. Application Layer - Layer7 (응용)

응용 계층은 사용자와 가장 가깝게 위치한 계층으로, 우리가 일반적으로 사용하는 웹 브라우저, 이메일 클라이언트, 메시징 앱 등의 응용 프로그램이 이 계층에서 동작한다. 이 계층에서는 사용자의 요청을 네트워크 서비스로 변환하거나, 네트워크에서 전달된 데이터를 **사용자가 이해할 수 있는 형태**로 제공한다.

## ✏️ Appendix

### Java, Spring과 어떤 관계가 있는지

서버 사이드 애플리케이션을 개발하면, 해당 애플리케이션은 네트워크 계층에서 데이터를 주고 받는다. 특히 응용 계층에서 클라이언트의 요청을 처리하고 응답을 반환하는 역할을 한다. 이 과정에서 전송 계층의 TCP/IP 프로토콜을 활용해 데이터를 안정적으로 전송하고, 세션 계층에서는 HTTP 세션을 관리하는 등 네트워크 계층의 여러 기능을 활용하게 된다.

1. **응용 계층(Application Layer)**: Java Spring에서는 컨트롤러(Controller)와 서비스(Service) 클래스가 이 계층에 해당한다. 사용자의 요청을 받아 처리하고, 응답을 다시 사용자에게 반환하는 역할을 한다.
2. **표현 계층(Presentation Layer)**: Java Spring에서는 뷰(View)가 이 계층에 해당된다. 데이터를 사용자가 이해할 수 있는 형태로 변환하거나, 사용자의 요청을 알맞은 형태로 변환하여 애플리케이션 계층에 전달한다.
3. **세션 계층(Session Layer)**: Java Spring의 세션 관리 기능이 이 계층을 담당한다. 사용자의 로그인 상태를 유지하거나, 특정 사용자의 요청을 인증하고 권한을 부여하는 역할을 한다.
4. **전송 계층(Transport Layer)**: Java Spring에서는 HTTP와 같은 프로토콜을 사용하여 데이터를 안전하게 전송한다. 이는 Spring MVC의 DispatcherServlet이 관리한다
5. **네트워크 계층(Network Layer)**: 이 계층은 보통 운영체제나 네트워크 장비가 담당하지만, Spring Cloud와 같은 라이브러리를 통해 마이크로서비스 간의 통신을 관리할 수 있다.
6. **데이터 링크 계층(Data Link Layer) 및 물리 계층(Physical Layer)**: 이 두 계층은 네트워크 하드웨어와 관련이 있다.
