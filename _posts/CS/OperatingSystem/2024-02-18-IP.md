# IP(Internet Protocol)

IP란, 고유한 식별자 역할을 하는 숫자 문자열이다.

### IP의 역할

IP는 데이터를 패킷이라는 작은 단위로 나누고, 이 패킷들을 목적지까지 전달하는 역할을 한다. 각 패킷에는 보내는 곳과 받는 곳의 IP 주소가 표시되어 있어, 패킷들이 올바른 경로를 따라 목적지까지 도달할 수 있게된다.

---

# ☺︎ IPv4와 IPv6

IPv4와 IPv6는 모두 인터넷 프로토콜(IP)의 버전이다. 이들은 컴퓨터나 스마트폰과 같은 디바이스가 인터넷에 연결되어 서로 통신할 때 사용하는 주소를 말한다. 두 가지 버전이 있는데, 구버전인 IPv4 그리고 지금 사용되는 IPv6이 있다.

### 예를 들면

IPv4를 생각해보면, 이는 집 주소와 비슷하다고 생각하면 쉽다. 예를 들어, '서울시 강남구 테헤란로 123'이라는 주소가 있다면, 이 주소를 통해 특정 집을 찾아갈 수 있듯이, IPv4 주소도 컴퓨터나 스마트폰을 찾아가는데 사용된다. 하지만 IPv4 주소는 숫자 네 덩어리로 이루어져 있어, '192.168.0.1'과 같은 형태를 가진다.

그런데 문제가 있다. IPv4 주소는 **약 43억 개밖에 만들 수 없어, 인터넷이 급속도로 성장하면서 주소가 부족**해지고 있다. 그래서 만들어진 것이 IPv6이다. IPv6 주소는 숫자와 알파벳을 조합해 만들어져, 이론적으로는 거의 무한대의 주소를 만들 수 있다. IPv6 주소는 '2001:0db8:85a3:0000:0000:8a2e:0370:7334'와 같은 형태를 가진다.

IPv6는 마치 도시가 발전하면서 길 이름에 번지 수뿐만 아니라 아파트 이름과 동, 호수까지 추가해야 하는 것과 비슷하다. 이렇게 하면 훨씬 더 많은 주소를 표현할 수 있게 된다.

# ☺︎ IPv4와 IPv6의 차이

| 근거 | IPv4 | IPv6 |
| --- | --- | --- |
| IP 주소의 크기 | IPv4는 32비트 IP 주소 | IPv6은 128비트 IP 주소 |
| 주소 지정 방법 | IPv4는 숫자 주소이며 이진 비트는 점(.)으로 구분 | IPv6은 이진 비트가 콜론(:)으로 구분되는 영숫자 주소 |
| 표시 방법 | 8비트씩 4부분으로 나눈 10진수 | 16비트씩 8부분으로 나눈 16진수 |
| 헤더 필드 수 | 12 | 8 |
| 주소 개수 | 약 43억개 | 2^128개 |
| 체크섬 | 체크섬 필드가 있음 | 체크섬 필드가 없음 |
| 예 | 12.244.233.165 | 2001:0db8:0000:0000:0000:ff00:0042:7879 |
| 주소 유형 | 유니캐스트, 브로드캐스트 및 멀티캐스트. | 유니캐스트, 멀티캐스트, 애니캐스트. |
| 수업 수 | IPv4는 다섯 가지 다른 기능을 제공한다. (A급부터 E급) | lPv6을 사용하면 IP 주소를 무제한으로 저장할 수 있다. |
| 보안 | IPSec 프로토콜 별도 설치 | 보안과 인증 확장헤더를 사용하여 보안 기능을 강화 |
