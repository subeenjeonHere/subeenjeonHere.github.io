# ☻ 기본 타입과 참조 타입

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/567e991f-89e5-4d0a-873b-1c45a3044964)

자바에서는 기본 타입(primitive type)과 참조 타입(reference type) 두 가지 종류의 타입을 사용한다.

기본 타입에는 **byte, short, int, long, float, double, char, boolean**이 있으며, 이들은 **실제 값을 저장**한다.

참조 타입에는 **배열, 열거, 클래스, 인터페이스**가 있으며, 이들은 **메모리의 주소를 저장**한다. 이 주소를 통해 실제 객체에 접근할 수 있다.

---

## ☺︎ 정수 타입

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/e3e6872d-029e-44d3-aad6-b1b564cfd43a)

기본으로 사용되는 데이터 타입은 **int형**이다.

단순히 정수 타입을 하나로 사용하지 않는 이유는, **메모리 상의 공간을 최적화**하기 위해서다.

예를 들어, 사과 16개를 박스 하나에 담으려고 한다면 사과 32개가 들어가는 박스를 사용할 때 공간 낭비가 발생한다. 따라서 이를 방지하기 위해, 필요한 만큼의 공간을 가진 적절한 정수 타입을 사용해야 한다.

---

## 컴퓨터에서 수의 표현

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/dd6b434d-76c5-4dec-b8f4-73900c09f457)

컴퓨터는 모든 데이터를 **이진수**(0과 1)로 표현한다.

여기서 **1바이트는 8비트로 구성**되어 있으며, 각 비트는 0 또는 1의 값을 가질 수 있다. 그러므로 8개의 비트가 있으면, 2^8, 즉 256의 경우의 수가 있다.

컴퓨터는 0부터 세기 시작하므로, 1바이트는 **0부터 255까지의 정수**를 표현할 수 있다.

그럼, int는 4바이트(32비트)이므로, 표현할 수 있는 값의 범위는 2의 32승이 된다. 따라서 int는 약 -21억에서 +21억까지의 정수 값을 표현할 수 있다.

---

## 음수 표현 방법

자바에서는 음수를 **2의 보수(complement)라는 개념**을 활용한다.

정수를 이진수로 변환하고 → 각 비트를 반전시킨 후, 1을 더하는 과정으로 음수를 표현하는데 이를 2의 보수라고 한다.

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/b947443a-cd42-4cc2-88dd-3c5b46f2d8a2)
### 음수를 표현하는 과정

| 단계 | 내용             | 이진수  |
|----|----------------|------|
| 1  | 양수 1을 이진수로 표현  | 0001 |
| 2  | 이진수의 각 비트를 반전  | 1110 |
| 3  | 반전된 이진수에 1을 더함 | 1111 |

따라서, -1은 이진수로 1111로 표현된다.

### 반전된 이진수에 1을 더하는 이유

원래의 양수 값을 정확히 반대되는 음수 값으로 표현하기 위해서다.

예를 들어, 1을 음수로 바꾸고 싶을 때, 1의 이진 표현은 0001이다. 이진수를 모두 반전시키면 1110이 되는데, 이는 -2를 의미한다. 따라서, 이 상태에서 1을 더하면 -1이 되고, 원래의 1을 정확히
반대되는 음수 -1로 표현할 수 있다.

## 기본 타입과 참조 타입의 저장 방식

기본 타입의 변수들은 **스택(Stack) 메모리 영역에 직접 값**을 저장한다.

반면에 참조 타입의 변수들은 **힙(Heap) 메모리 영역에 생성된 객체의 번지 값**을 갖고 있다.

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/076ba45d-c5fd-4be4-97f4-ceee0718f781)
### 스택 (Stack) 영역

**기본 타입의 데이터와 참조 타입의 주소 값**이 저장된다.

이 영역에서 데이터의 생성과 소멸은 **스택의 푸시(push)와 팝(pop) 동작**으로 이루어지며, 이 동작들은 CPU에 의해 빠르게 처리된다. 따라서, 작고 빠르게 사용하고 소멸하는 데이터를 스택 영역에
저장한다.

### 힙 (Heap) 영역

반대로, 힙 영역은 **객체가 생성될 때 메모리 공간이 할당**되고, 가비지 컬렉터에 의해 객체가 소멸될 때 메모리 공간이 회수된다.

## 참조 변수의 ==, != 연산

기본 타입 변수의 ==, != 연산은 **변수 값이 같은지, 아닌지**를 조사하지만, 참조 변수의 ==, != 연산은 **동일한 객체를 참조하는지, 다른 객체를 참조하는지** 알아볼 때 사용된다.

결국 참조 타입에서 **== 연산은** **메모리 주소 값을 비교하는 것을** 의미한다.

따라서 동일한 객체를 참조하고 있다면 == 연산 결과는 true, ≠ 연산 결과는 false가 된다.

### 그래서 인텔리제이가

이전 코딩테스트 풀 때 String 타입을 == 연산자를 사용했었는데, 인텔리제이에서 `equals` 메소드를 추천(?) 해줬다.

String은 참조 타입이기에, String의 내용 자체(내부 문자열)를 비교하려면 `equals` 메소드를 사용해야 한다.

반면, int는 기본 타입이므로 `==` 연산자를 사용하면 실제 값 자체를 비교하게 된다.

## null과, nullPointerException

1. 참조 타입 변수는 **힙 영역의 객체를 참조하지 않는다는 뜻**으로 **null** 값을 가질 수 있다.
    1. 참조 타입 변수는 **객체를 가리키는 변수다.** 그런데 해당 객체는 아무런 객체를 가리키지 않을 수도 있다. 이때 변수는 **null** 값을 가지게 된다.
2. **null** 값도 **초기값으로** 사용할 수 있기 때문에 null로 초기화된 참조 변수는 스택 영역에 생성된다.
    1. 참조 타입 변수를 사용할 때, 초기화를 하지 않으면 해당 변수는 자동으로 **null로 초기화된다**.
3. 참조 타입 변수가 null 값을 가지는지 확인하려면 ==,!= 연산을 사용하면 된다.

![image](https://github.com/subeenjeonHere/subeenjeonHere.github.io/assets/145312273/29cc8cf6-5825-44e7-bf66-cee2ef194afc)

### 그럼, null일거면 왜 생성하는 것인지

- 나중에 값이 할당될 것으로 예상되는 경우
- 변수 선언 시, 이미 객체를 생성할 수 있는 상황이 아니거나 나중에 할당할 준비를할 때 초기값으로 Null을 할당한다.

근데 내가 알고리즘 문제를 풀 때 그렇게 하고 있었다. 예를 들면

```java
import java.util.*;

public class DFSExample {
    private static Stack<Integer> stack; // 스택 필드를 null로 선언

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        stack = new Stack<>(); // 스택 객체를 생성하여 초기화
    }
}
```

1. null로 선언된 필드는 **스택 영역**에 할당되고

```java
private static Stack<Integer> stack;
```

1. 해당 필드가 참조할 객체는 필요한 시점에 **메인 메소드에서 힙 영역**에 생성된다.

```java
 stack =new Stack<>();
```

### 왜 기본 타입은 스택 영역에 직접 데이터를 저장하고, 참조 타입은 힙 영역에 객체를 생성하는 건지

일반적으로 간단한 데이터를 나타내며, 그 크기가 일정하다. 이러한 기본 타입 데이터는 스택 메모리에 직접 값을 저장한다.

반면 참조 타입은 **복잡한 데이터, 객체를 표현할 때 사용**된다. 이러한 데이터 크기는 가변적이거나, 큰 경우가 많다. 또한, 참조 타입 변수는 해당 **객체의 메모리 주소를 가리키며, 실제 데이터는 힙 메모리
영역에 저장된다.**

### 저런 이유로 스택에 저장할 수 없는 이유가 뭔지

스택 메모리는 **크기가 작고 제한된 공간**으로, 메모리의 **할당과 해제가 자동**으로 이루어 진다. 기본 타입의 데이터는 크기가 일정하며, 처리 속도가 빠르기 때문에 스택 메모리에 적합하다.

반면 참조 타입은 **데이터 크기가 가변적이거나 크며, 데이터의 생명 주기 관리가 필요하다. 이로써 복잡한 데이터는 스택보다, 큰 공간을 가진 힙 영역에 저장하는 것이 더 효율적이다.**

또한, 참조 타입은 여러 변수가 동일한 객체를 참조할 수 있도록 하기 위해 힙 메모리에 저장되며, 메모리 공간을 효율적으로 절약할 수 있다.

---

### Integer, int 차이

- 기본 타입 int는 **null 값을 가질 수 없다**. 반면, Integer는 참조 타입이므로 **null 값을 가질 수 있다.**
- **컬렉션 프레임워크 사용:** 객체만을 저장할 수 있다. 기본 타입인 int는 객체가 아니기 때문에, 컬렉션에 직접 저장할 수 없다. 따라서 Integer와 같은 래퍼 클래스를 사용하여 기본 타입을 객체로
  변환해야 한다.
- **Autoboxing과 Unboxing**: 기본 타입, 래퍼 클래스 간의 변환을 자동으로 처리해주는 자바의 기능이다.

### 오토박싱 (AutoBoxing)

기본 타입을 → 래퍼 클래스 객체로 자동으로 변환하는 과정

```java
int primitiveInt = 10;
Integer wrapperInteger = primitiveInt; // 오토박싱
```

### 언박싱 (Unboxing)

래퍼 클래스 객체를 → 기본 타입으로 자동 변환하는 과정

```java
Integer wrapperInteger = 10;
int primitiveInt = wrapperInteger; // 언박싱
```
